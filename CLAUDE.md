# PDR-I QA Toolkit

Risk-driven QA and defect prediction system for the PowerDirector iOS app (`com.cyberlink.powerdirector`). It fetches bug data, scores modules by risk, clusters bugs by theme, predicts defects, auto-generates test skeletons, and visualizes everything via a Streamlit dashboard.

## Tests

All files in `tests/` are placeholder skeletons generated by `auto_tag_tests.py` (stage 7 of the data pipeline). They define the class/method structure and Allure/pytest metadata but contain only `pass` stubs — none are implemented. Do not attempt to run or extend these tests as functional code.

## Pipeline Commands

```bash
# Run full data refresh pipeline
./refresh_pipeline.sh

# Faster partial runs
./refresh_pipeline.sh --skip-ollama    # Use heuristic scorer instead of Ollama
./refresh_pipeline.sh --skip-cluster   # Skip NLP clustering (~5 min savings)
./refresh_pipeline.sh --skip-predict   # Skip ML defect predictions
./refresh_pipeline.sh --dry-run        # Print steps without executing
```

## Architecture

### Data Pipeline (`scripts/`)

Scripts run in this fixed order — each stage outputs CSV/JSON files consumed by the next:

1. `fetch_from_n8n.py` — fetch raw bugs from n8n webhook
2. `parse_ecl_export.py` — parse Excel/JSON → structured `ecl_*.csv`
3. `compute_risk_scores.py` — compute risk metrics → `risk_register_*.csv`
4. `ai_risk_scorer.py` — AI scoring via heuristic / Ollama / Claude API
5. `cluster_bugs.py` — NLP clustering → `*_clustered.csv`, `*_cluster_summary*.csv`
6. `predict_defects.py` — ML forecasting → `*_predictions.csv`, `*_by_cluster.csv`
7. `auto_tag_tests.py` — generate test skeletons in `tests/generated/test_<module>.py`

`refresh_pipeline.sh` orchestrates all stages with atomic file promotion: writes to `data/staging/`, validates, then `mv` to `data/`.

`bug_heatmap_dashboard.py` is the Streamlit UI (9 tabs) reading all pipeline outputs. It runs as a persistent LaunchAgent on the Mac Mini at `http://<hostname>:8501`.

## Key Conventions

### Data File Naming

| Pattern | Stage |
|---|---|
| `ecl_*.csv` | Parsed bug data |
| `risk_register_*.csv` | Risk scoring output |
| `*_clustered.csv` | Bug clustering output |
| `*_predictions.csv` | Defect forecasts |
| `*_pending.json` | Unconfirmed module mappings (fuzzy-match candidates) |
| `quadrant_summary.md` | P1–P4 module prioritization table |

### Appium Driver Config

Configured via environment variables (with defaults):

| Variable | Default |
|---|---|
| `APPIUM_URL` | `http://127.0.0.1:4723` |
| `BUNDLE_ID` | `com.cyberlink.powerdirector` |
| `DEVICE_NAME` | `iPhone` |

Driver has 10-second implicit wait and `no_reset=True` (preserves app state between tests). Session-scoped — one driver instance for the entire test run.

### CI/CD (Jenkins)

The `Jenkinsfile` runs on `mac-mini-m1` agent. `TEST_SCOPE` parameter controls which quadrant stages execute. All stages use `|| true` — failures don't block subsequent stages. Allure results are published from `allure-results/` and visual diffs are archived from `visual_results/`.

## Environment

- Python 3.14 in `.venv/` (arm64 native on M1)
- `setup_mac_mini.sh` for one-time environment setup
- Ollama is optional; `--skip-ollama` uses heuristic scoring instead
